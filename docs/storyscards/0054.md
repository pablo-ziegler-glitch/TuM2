## Alcance implementado

- Magic link email end-to-end con listener real de deep links (`app_links`).
- Resolución same-device: procesamiento automático del link con `pending_email_link`.
- Resolución cross-device: guardado de link pendiente + navegación a `/email-verification?cross_device=true`.
- Google Sign-In mantenido operativo en flujo mobile/web según plataforma.
- Router con `pendingRoute` real para deep links pre-auth y consumo post-auth.
- Guards consistentes por auth/rol con soporte de query params.
- Hardening de `signOut`: limpieza de SharedPreferences y providers de estado local de auth/router.
- Hardening de analytics AUTH: eventos safe/no-op si Firebase Analytics no está inicializado.

## Decisiones técnicas

- Se introdujo `deep_link_listener.dart` como listener global desacoplado, activado desde `main.dart`.
- Se creó `pending_route_provider.dart` para compartir `pendingRoute` entre router y auth sin acoplar módulos.
- Se inyectaron dependencias de autenticación (`AuthClient`, `GoogleSignInClient`) para tests determinísticos sin Firebase real.
- `RouterGuards` ahora normaliza path desde `location` para evitar regresiones en rutas públicas con querystring.
- Prioridad de seguridad en owner sin onboarding: onboarding obligatorio antes de restaurar pending route.
